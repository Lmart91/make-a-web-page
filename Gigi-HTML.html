5 ways of Thinking

One of the most important things when you start to learn code is begin to think like a programmer.
This way of thinking is divided, as you can guess from the title, into 5 ways of thinking:

  1. Procedural Thinking
    It means creating perfectly clear and unambiguous instructions for a computer to follow.
  2. Abstract Thinking
    Find similarity between apparently different things.
  3. Systematic Debugging
    Is a systematic process to identify and track down potential causes of a computer program that doesn't work until to find the problem and fix it.
  4. Technological Empathy
    It means start to think in the way the computer would (e.g. when you see <b>Hello!</b> the first thing you should think about is that Hello! is bold !).
  5. Systems Thinking
    Take a big problem and break the solution into a set of steps. Programmers do this when they create a plan (often on paper) for how a program will work.

HTML

HTML is the acronym of HyperText Markup Language and it is made up of:

  • Text content → what you see
  • Markup → what it looks like
  • References to other documents → e.g. images and videos
  • Links → to other pages

The text content is literally the text as it is written.

The Markup is made up of tags, the opening and the closing tag that together the text content they form an element.
e.g. <b>contents</b> is a tag to make a text bold, <em>content</em> is to make a text italic.

Attributes are used to display an image or link to a video or a website.

The anchor tag is used to link a website or a video to a text <a href="link"> derp</a> . Just click on Udacity.


The img tag is used to display images <img src="url" alt="text">
This is a void tag, a tag that has no content and it doesn't need a closing tag.

Whitespace

It is the space between multiple lines created by:

  • <br>, break tag (void tag).
  • <p>contents</p>, paragraph tag

To understand why we have two ways of making lines, we need to talk about:

  • Inline
  • Block

Inline tags, like <br>, <a>, <img> and <span>, simply create a lines.
Block tags like <p> and <div> instead create a box, that surrounds the text, which has a height and width.

CSS

We know that document written in HTML defines the structure and content of the page but a webpage just with text seems a bit older.
To define the style we have to use another language called CSS.

CSS (Cascading Style Sheets) is a style sheet language that describe the look and the format of a document written in a markup language like the HTML language.

Everything is a box

All the elements of a HTML document are small boxes within boxes bigger and bigger.
These boxes are created in the HTML language using <div>content</div>, this is called div tag.

Adding CSS Style to HTML Structure

Avoid Repetition !

Avoiding repetition is really important when writing a HTML structure so that is possible to avoid errors and apply style to a lot of element with a single CSS line. If a programmer doesn't avoid repetition, he will have to do the same thing again and again and sooner or later will make an error (or a lot of errors!). Therefore avoid repetition for you and for those who read your code !

Adding Style

First of all is necessary to add a single line inside the <head></head> tag to link your HTML structure to the CSS style file, in this way:
<link rel="stylesheet" href="style.css"> (where style.css is replaced with the name of the file that contains style information)

With CSS the most specific rule is applied to every element, this concept is what the word "Cascading" means.
Rules are applied not just to the elements which are directly matched, but also to all of their child elements. This is a mechanism called Inheritance and descendant elements can inherit CSS property values from any ancestor element enclosing them. Inheritance allows to avoid repetition of writing certain properties in a style-sheet and so to write less CSS.

Styling up

In CSS to add style, you must first write "a selector" to define what element of the HTML structure will have that style.

The Box Model

Everything on a webpage is a box

  • Content
    The content is the image or text that appears on the website.
  • Padding
    The padding clears an area around the content and it's affecting by the background-color of the box.
  • Border
    The border goes around the padding and the content. It is inherited from the color property of the box.
  • Margin
    The margin clears an area around the border and doesn't have a background color, it's completely transparent.
